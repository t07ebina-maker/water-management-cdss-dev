# AIアシスタント向け開発ルール（汎用テンプレート）

このファイルはClaude Code・Codex等のAIツールが作業開始時に読み込むルールブックである。
すべての指示はこのファイルの内容に従うこと。

---

## 1. 新規プロジェクト作成時のファイル構成ルール

新しいWebアプリを作成する場合、必ず以下の3ファイル構成で作成すること。
1つのファイルにまとめて作成することを禁止する。

    index.html   … HTML構造のみ（骨組み・要素の配置・階層）
    style.css    … CSSのみ（見た目・色・レイアウト・フォント）
    main.js      … JavaScriptのみ（計算・判定・イベント処理・ロジック）

index.html の末尾には必ず以下のように外部ファイルを読み込むこと。

    <link rel="stylesheet" href="style.css">
    <script src="main.js" defer></script>

---

## 2. 修正依頼時の自動判別ルール

修正内容に応じて、操作するファイルを自動判別し、該当ファイルのみを変更すること。
関係のないファイルを勝手に変更することを禁止する。

    ロジック・計算・判定・データ処理・DOM操作の変更  →  main.js のみ操作
    見た目・色・レイアウト・フォントの変更  →  style.css のみ操作
    画面構造・要素の追加削除・階層の変更    →  index.html のみ操作

複数ファイルにまたがる変更が必要な場合は、作業前に必ず報告してから着手すること。

---

## 3. index.html の保護ルール（GitHub Pages運用時）

GitHub Pages で公開中のプロジェクトでは以下を厳守すること。

- 作業対象ファイルは index_work.html を使用する
- index.html は読み取り専用とし、変更・上書き・削除を禁止する
- 作業完了・動作確認後、人間の承認を得てから index.html に反映する
- 上記制約を守れない場合は作業を中断して報告すること

---

## 4. ファイルバージョン管理ルール

- 成果物ファイルは上書きせず、必ずバージョン番号（v1・v2…）をファイル名に付与する
- 過去バージョンは削除せず保持する
- 作業前に git commit で現在の完成版を保護すること

    git add .
    git commit -m "vXX 完成版 確定"

---

## 5. 報告ルール

以下の場合は必ず作業前に報告し、確認を得てから進むこと。

- 複数ファイルにまたがる変更が発生する場合
- index.html の変更が必要と判断した場合
- ルールに従うと要件を満たせない場合
- エラーが発生した場合は変更を即時停止し、エラー内容・原因・対象ファイルを報告すること

---

## 6. 禁止事項まとめ

- index.html 内への style タグ・script タグの直接記述
- 1つのファイルへのHTML・CSS・JS混在
- 指示されていないファイルの変更
- 確認なしの index.html 上書き
